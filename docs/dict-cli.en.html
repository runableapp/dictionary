dict-cli Complete User Manual

An offline SQLite dictionary CLI with two optional runtime modes: REST API server (serve) and MCP stdio server (mcp). This page provides full usage guidance in one place.

  • Offline dictionary lookup
  • REST API serving mode
  • MCP tooling integration

D I C T - C L I   M A N U A L

1) Features

  • SQLite DB file/directory based dictionary queries
  • Search commands: exact, prefix, meaning
  • Runtime modes: local CLI / HTTP REST API / MCP (stdio)

2) Requirements

  • Go 1.25+
  • A dictionary DB path (built separately)
  • Unified DB file: /path/to/dictionary_fts5.db
  • Or per-dictionary DB directory: /path/to/dicts

3) Build (Taskfile, static linking)

From the dictionary-cli-app directory:

./build.sh

build.sh invokes the Taskfile build target:

task build

Build command (static-link friendly):

CGO_ENABLED=0 go build -trimpath -ldflags "-s -w" -o bin/dictcli ./cmd/dictcli

Optional check on Linux:

ldd ./bin/dictcli

If output includes not a dynamic executable, it is statically linked.

4) Test

Validate CLI behavior against DB files under dictionary-data/dicts:

./test.sh

test.sh runs:

  • smoke test via scripts/demo_cli_usage.sh
  • sanity checks for every dicts/*.db using groups

5) CLI Run Examples

./bin/dictcli --db /absolute/path/to/dictionary-data/dictionary_fts5.db groups
./bin/dictcli --db /absolute/path/to/dictionary-data/dicts --group ko-en exact --term 사전
./bin/dictcli --db /absolute/path/to/dictionary-data/dicts --group en-ko prefix --term dict
./bin/dictcli --db /absolute/path/to/dictionary-data/dicts --group ko-en meaning --query dictionary

Output modes:

./bin/dictcli --db /absolute/path/to/dictionary-data/dicts --group ko-en exact --term 옥편 --output rich
./bin/dictcli --db /absolute/path/to/dictionary-data/dicts --group ko-en exact --term 옥편 --output plain
./bin/dictcli --db /absolute/path/to/dictionary-data/dicts --group ko-en exact --term 옥편 --output md
./bin/dictcli --db /absolute/path/to/dictionary-data/dicts --group ko-en exact --term 옥편 --output json
./bin/dictcli --db /absolute/path/to/dictionary-data/dicts --group ko-en exact --term 옥편 --output xml

6) Configuration

The config file is optional (see config.example.ini).

DB path resolution priority:

  1. --db <path>
  2. DICTCLI_DB_PATH
  3. [database] path in config.ini

Config file path priority:

  1. --config <path>
  2. DICTCLI_CONFIG
  3. ./config.ini in current directory

7) REST API (Full Usage)

Start server:

./bin/dictcli \
  --db /absolute/path/to/dictionary-data/dicts \
  --group ko-en \
  --limit 20 \
  --timeout 60 \
  serve --host 127.0.0.1 --port 8080

Endpoints:

  • GET /healthz
  • GET /groups
  • GET /exact
  • GET /prefix
  • GET /meaning

Common query parameters:

  • group: dictionary group ID (e.g. ko-en, en-ko)
  • limit: max result count
  • timeout: timeout in seconds
  • source_prefix: source_dict prefix filter (e.g. lingoes)

Required per endpoint:

  • /exact: term + group
  • /prefix: term + group
  • /meaning: query + group

/meaning specific parameter:

  • ranking=true|false (default false)
  • false: fast mode, true: ranked mode (slower, more precise)

Request examples:

curl -s "http://127.0.0.1:8080/healthz"
curl -s "http://127.0.0.1:8080/groups"
curl -s --get --data-urlencode "group=ko-en" --data-urlencode "term=가게" --data-urlencode "limit=3" "http://127.0.0.1:8080/exact"
curl -s --get --data-urlencode "group=en-ko" --data-urlencode "term=dict" --data-urlencode "limit=5" "http://127.0.0.1:8080/prefix"
curl -s --get --data-urlencode "group=ko-en" --data-urlencode "query=dictionary" --data-urlencode "limit=5" --data-urlencode "ranking=false" "http://127.0.0.1:8080/meaning"

Success response example:

{
  "results": [
    {
      "group": "ko-en",
      "headword": "사전",
      "source_dict": "stardict-koreanenglish",
      "overlap_count": 2,
      "definition": "dictionary ...",
      "duplicates": [
        {
          "source_dict": "stardict-koreanenglish",
          "definition": "dictionary ..."
        }
      ]
    }
  ]
}

Error response example:

{
  "error": "exact requires query params: term, group"
}

/groups response example:

{
  "groups": ["en-ko", "ko-en", "ko-ko"]
}

/healthz response example:

{
  "status": "ok"
}

8) MCP Server (Detailed)

Start MCP server:

./bin/dictcli \
  --db /absolute/path/to/dictionary-data/dicts \
  --group ko-en \
  --limit 20 \
  --timeout 60 \
  mcp --name dictcli --version 0.1.0

Exposed MCP tools:

  • groups (no args, response { "groups": [...] })
  • exact (required term, response { "results": [...] })
  • prefix (required term, response { "results": [...] })
  • meaning (required query, response { "results": [...] })

MCP client config example file: manual/mcp_client_config.example.json

9) Ubuntu systemd Service

Added files:

  • Service unit: deploy/systemd/dictcli-api.service
  • Launcher script: scripts/run_dictcli_api.sh

1) Deployment path example:

sudo mkdir -p /opt/dictcli/bin /opt/dictcli/scripts
sudo cp ./bin/dictcli /opt/dictcli/bin/dictcli
sudo cp ./scripts/run_dictcli_api.sh /opt/dictcli/scripts/run_dictcli_api.sh
sudo chmod +x /opt/dictcli/scripts/run_dictcli_api.sh

2) Service user example:

sudo useradd --system --no-create-home --shell /usr/sbin/nologin dictcli || true
sudo chown -R dictcli:dictcli /opt/dictcli

3) Create environment file /etc/default/dictcli:

DICTCLI_BIN=/opt/dictcli/bin/dictcli
DICTCLI_DB_PATH=/home/keith/workspace/dictionary/dictionary-data/dicts
DICTCLI_HOST=127.0.0.1
DICTCLI_PORT=8080
DICTCLI_GROUP=ko-en
DICTCLI_LIMIT=20
DICTCLI_TIMEOUT=60
# DICTCLI_SOURCE_PREFIX=lingoes

4) Register and start service:

sudo cp ./deploy/systemd/dictcli-api.service /etc/systemd/system/dictcli-api.service
sudo systemctl daemon-reload
sudo systemctl enable --now dictcli-api
sudo systemctl status dictcli-api
